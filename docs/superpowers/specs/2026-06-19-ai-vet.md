# AI Vet — "Dobby"

## Summary

Add a streaming AI chat feature powered by Google Gemini 2.0 Flash (free tier via Google AI Studio). The agent is named "Dobby", greets the user by name, speaks English and European Portuguese, is scoped strictly to dog health questions (guardrails for off-topic), and shows a non-professional disclaimer. Chat is session-only (no persistence). The AI receives the dog's live health context (weight, food, medications, vaccinations) on every request.

Navigation change: AI Vet replaces Vet Visits in the bottom/sidebar nav. Vet Visits moves to the More page.

## User flow

1. User taps the **AI Vet** tab (bot icon) in the nav → `/ai-vet`
2. Welcome screen shows: "Hey [name]! I'm Dobby, your AI vet" + capability bullets + disclaimer banner
3. User types a question → streams back a response from Gemini
4. Off-topic question → guardrail redirect ("I'm a virtual vet, I can't help with that…")
5. Language follows the app locale (EN/PT) — both the UI and the AI respond in the user's language

## Commit 5 — `feat: add streaming chat api route for ai vet`

Create `app/api/chat/route.ts`.

- Auth-gated: returns 401 if no user
- Accepts `{ messages, context }` in the POST body
- Builds a system prompt with the dog's health context and language instruction
- Guardrail is enforced in the system prompt (off-topic → redirect response)
- Streams via `streamText` from `ai` + `google('gemini-2.0-flash')` from `@ai-sdk/google`

```typescript
import { streamText } from 'ai';
import { google } from '@ai-sdk/google';
import { createClient } from '@/lib/supabase/server';

export interface DogContext {
  puppyName: string;
  breed: string;
  age: string;
  sex: string | null;
  weight: string | null;
  food: string | null;
  medications: string | null;
  vaccinationStatus: string | null;
  locale: string;
}

export async function POST(req: Request) {
  const supabase = await createClient();
  const { data: { user } } = await supabase.auth.getUser();
  if (!user) return new Response('Unauthorized', { status: 401 });

  const { messages, context }: { messages: Parameters<typeof streamText>[0]['messages']; context: DogContext } = await req.json();
  const { puppyName, breed, age, sex, weight, food, medications, vaccinationStatus, locale } = context;

  const language = locale === 'pt' ? 'European Portuguese (Portugal)' : 'English';

  const systemPrompt = `You are "Dobby", a friendly and knowledgeable AI veterinary assistant for a ${breed} named ${puppyName}.

Your rules:
- ONLY answer questions about dog health, nutrition, behaviour, care, and wellness.
- If the user asks about something unrelated (e.g. restaurants, travel, news, anything non-dog-health), respond with exactly: "Hey, I'm a virtual vet — I can't help with that! But I can help you with things related to ${puppyName}, like vaccinations, nutrition, health concerns, behaviour, and more."
- You are NOT a real veterinarian. For serious or urgent health concerns, always recommend consulting a qualified vet in person.
- Be warm, friendly, and concise.

${puppyName}'s current profile:
- Breed: ${breed}
- Age: ${age}
- Sex: ${sex ?? 'Unknown'}${weight ? `\n- Latest weight: ${weight}` : ''}${food ? `\n- Current food: ${food}` : ''}${medications ? `\n- Active medications: ${medications}` : ''}${vaccinationStatus ? `\n- Vaccination status: ${vaccinationStatus}` : ''}

Always respond in ${language}.`;

  const result = streamText({
    model: google('gemini-2.0-flash'),
    system: systemPrompt,
    messages,
  });

  return result.toDataStreamResponse();
}
```

## Commit 6 — `feat: add ai vet page (server shell + streaming chat ui)`

Create two files: server shell and client chat component.

### `app/ai-vet/page.tsx` (server component)

- Auth guard (redirect to `/login` if no user, `/profile/setup` if no membership)
- Fetches in parallel via `Promise.all`:
  - `puppies` — name, breed, date_of_birth, sex (use `calculateAge` from `lib/utils.ts` for age string)
  - `weight_entries` — latest 1, ordered by date desc
  - `food_entries` — current (where `end_date is null`), ordered by `start_date` desc, limit 1
  - `medications` — active (where `end_date is null` or `end_date > today`), limit 3

## Out of scope

- Chat history persistence (session-only via `useChat`)
- Image or file uploads in chat
- Rate limiting (free Gemini tier handles 1,500 req/day — sufficient for personal use)
- Streaming cancellation / abort
